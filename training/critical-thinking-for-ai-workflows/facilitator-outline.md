# Facilitator Outline

## Module Positioning

**From Evidence to Decision: Critical Thinking for AI-Assisted Analysis and Process Governance** is a public workshop-style module for discussing how teams should review AI-assisted analytical outputs before using them in records, decisions, or follow-up actions.

This is not an accredited course, certification, official training program, production readiness claim, or implementation manual.

## Audience

This outline is suitable for portfolio discussions, internal learning conversations, AI transformation workshops, process governance reviews, and operating model design sessions.

## Suggested Session Length

- Short version: 45 minutes
- Standard version: 90 minutes
- Extended version: 2 hours with group exercises

## Learning Outcomes

By the end of the session, participants should be able to:

- Separate source-backed facts from interpretation and assumptions.
- Identify open questions, risks, decisions, and follow-up actions.
- Explain why evidence mapping improves reviewability.
- Explain why validation gates do not make output true or approved.
- Apply a review gate before using AI-assisted output.
- Describe how human review creates an approved record or action.

## Agenda

| Time | Section | Purpose |
| --- | --- | --- |
| 5 min | Framing | AI output is a reviewable signal, not final authority. |
| 10 min | Core distinctions | Define fact, interpretation, assumption, open question, risk, decision, and action. |
| 15 min | Evidence mapping | Show how claims become reviewable through source links and confidence boundaries. |
| 15 min | Validation vs approval | Explain what structural validation can and cannot prove. |
| 20 min | Group exercise | Classify synthetic source notes using the evidence vs interpretation example. |
| 15 min | Review gate | Apply the AI output review gate to decide whether output is ready for handoff. |
| 10 min | Governance connection | Connect the pattern to The Daltons, SAMAEL, NOESIS, and the Black Signal Governance Model. |

## Facilitation Notes

- Keep the discussion focused on review discipline, not tool performance.
- Ask participants to justify claims with source evidence.
- Treat uncertainty as useful information, not a defect to hide.
- Reinforce that validation is a gate, not approval.
- Reinforce that approval requires an accountable human reviewer.
- Avoid turning risks into recommendations before review.
- Avoid turning open questions into invented answers.

## Suggested Exercise Flow

1. Give participants the synthetic source notes in the evidence-vs-interpretation example.
2. Ask them to classify each statement into one category.
3. Ask what evidence supports each source-backed fact.
4. Ask which items require human confirmation.
5. Ask whether the material is ready for a decision, a follow-up action, or neither.
6. Apply the AI output review gate.
7. Discuss what would be required before an approved record could be created.

## Debrief Questions

- Which statements felt factual but were actually interpretations?
- Which assumptions were useful but still unconfirmed?
- Which open questions should block approval?
- Which risks were descriptive rather than recommendations?
- What would a human reviewer need before approving an action?
- Where would validation help, and where would it be insufficient?

## Public-Safe Boundary

Use only synthetic or sanitized examples. Do not use personal data, company confidential data, real meeting transcripts, real project documents, real task logs, internal URLs, private paths, credentials, tokens, secrets, production configuration, or private infrastructure details.
