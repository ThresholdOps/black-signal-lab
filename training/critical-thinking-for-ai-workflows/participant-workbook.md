# Participant Workbook

## Module

**From Evidence to Decision: Critical Thinking for AI-Assisted Analysis and Process Governance**

This workbook supports a public-safe workshop on reviewing AI-assisted outputs before they become records, decisions, or actions.

## Key Principle

AI output is a reviewable signal, not final authority.

## Exercise 1: Classify the Statement

Use the categories below to classify each statement in a synthetic artifact.

| Category | What It Means |
| --- | --- |
| Source-backed fact | Directly supported by supplied source material. |
| Interpretation | A reasoned reading of source material. |
| Assumption | Plausible but not proven. |
| Open question | A gap that still needs an answer. |
| Risk | A possible adverse outcome or uncertainty. |
| Decision | An approved choice by an accountable reviewer or body. |
| Follow-up action | Work accepted for someone to perform after review. |

Questions to ask:

- What source supports this?
- Is the statement directly stated or inferred?
- What is still unconfirmed?
- Does this require an accountable decision?
- Is this an action, or just a possible next step?

## Exercise 2: Build a Minimal Evidence Map

For each claim, record:

| Claim | Source Evidence | Category | Confidence | Open Question |
| --- | --- | --- | --- | --- |
| `[claim]` | `[source reference]` | `[category]` | `[high / medium / low]` | `[question or none]` |

Review prompts:

- Does every claim have source support?
- Are interpretations labeled?
- Are assumptions separated from facts?
- Are open questions preserved?
- Are risks described without becoming recommendations?

## Exercise 3: Review AI Output Before Use

Before using an AI-assisted artifact, check:

- Does it stay inside the input boundary?
- Does it distinguish evidence from interpretation?
- Does it preserve uncertainty?
- Does it avoid unsupported facts?
- Does it avoid making decisions?
- Does it avoid assigning work without approval?
- Does it avoid recommendations unless explicitly requested and reviewed?
- Does it require human review before becoming a record or action?

## Exercise 4: Worked AI Output Review

Read the [Worked AI Output Review](examples/worked-ai-output-review.md). Use the classification categories, decision-readiness checklist, and AI output review gate to identify which parts of the flawed summary are source-backed, interpreted, assumed, unresolved, risky, unsupported, or incorrectly presented as approved.

Then compare your reasoning with the facilitator answer key if it is being used in a guided session.

## Validation vs Approval

Validation can show that an artifact is structured, complete enough to inspect, and free from obvious boundary violations.

Validation cannot prove that:

- the output is true
- the output is complete
- the output is approved
- the output should trigger action
- the output is safe for operational use

Approval requires human review by an accountable reviewer or group.

## Reflection

Answer these after the exercise:

- Which item was easiest to classify?
- Which item was most ambiguous?
- What evidence would improve confidence?
- Which open question should block approval?
- What would need to happen before a follow-up action becomes accepted?
